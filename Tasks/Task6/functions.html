Структура работы приложения и Базовые функции для работы с Kivy

№ п/п Основные функции и модули для работы приложения Роль в разработке приложения
1 Модуль kivy.app.App Необходим для запуска инициализации приложения
2 Модуль kivy.uix.NLayout Под N подразумевается тип лайаута. Необходим для компоновки виджетов. Можно выделить 4 основных типа лайаута: GridLayout, RelativeLayout, FloatLayout и BoxLayout
3 Модуль kivy.uix.Widget Используется в классах, отдельных от основного, чтобы присвоить классу значение виджета
4 Модуль kivy.uix.Label Используется в качестве отображения текста. Основные параметры: text [str], pos_hint [dict], font_size [int], bold [bool], font_name [str]
5 Модуль kivy.uix.Button Используется в качестве отображения кнопки. Основные параметры: text [str], pos_hint [dict], size_hint [tuple], color [str], bold [bool], background_normal, background_down, size [tuple], pos_x [int/float], pos_y [int/float], pos [tuple]
6 Модуль kivy.Config Необходим для конфигурации приложения: присвоение размера экрана, полноэкранного режима и т.п.
7 Модуль kivy.animation.Animation Нужен для присвоения анимаций виджету
8 Функция bind() Необходима для присвоения кнопки возможности выполнить определенную часть кода.
Базовые параметры данной функции: on_press [self.def], on_release [self.def]
9 Функция add_widget (self.X.add_widget) Необходима для добавления виджета на экран приложения. X - ваш лайаут, если вы хотите добавить виджет на сам экран, вне лайаута - используйте self.add_widget(self.ваш_виджет)
10 Экран (ПР: class TestScreen(Screen, RelativeLayout) Одна из "Страниц" приложения, которой можно присвоить параметр Экрана, чтобы он им являлся, тип компоновки виджетов на нем и др.
